ANALISE: NOTIFICACOES DISCORD PARA NOTAS E SESSOES

Sim, e possivel conectar o painel com o Discord.

A forma mais simples e adequada para este projeto e usar um Discord Webhook. O webhook fica salvo como variavel de ambiente no servidor, por exemplo:

DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/...

Essa URL nunca deve ir para o navegador. Ela deve ser usada apenas dentro das funcoes server-side em src/lib/api/campaign.functions.ts.


COMO FUNCIONARIA

Hoje o painel salva o conteudo inteiro da campanha pela funcao saveCampaignContent. Para detectar novidades, o servidor pode:

1. Ler o conteudo atual do banco antes de salvar.
2. Comparar com o conteudo novo enviado pelo painel.
3. Identificar sessoes novas e notas novas.
4. Salvar o conteudo no Supabase.
5. Depois do salvamento bem-sucedido, enviar a mensagem para o Discord.

Esse fluxo e importante porque evita notificar o Discord sobre algo que falhou ao salvar.


O QUE PODE SER NOTIFICADO

Nova nota:

"Nova nota adicionada em Tenebre: <titulo>"

Pode incluir:
- titulo
- data/contexto
- link para /notas

Nova sessao:

"Novo resumo de sessao adicionado: Sessao XX - <titulo>"

Pode incluir:
- numero da sessao
- titulo
- data
- link para /sessoes/<slug>


MELHOR CRITERIO PARA DETECTAR NOVOS ITENS

Para sessoes:
- considerar nova sessao quando o number ainda nao existia antes; ou
- considerar nova sessao quando o slug ainda nao existia antes.

Eu recomendo usar number para sessoes, porque ele ja e unico no banco.

Para notas:
- hoje as notas usam title + date como chave unica.

Entao a deteccao pode ser:
- nota nova = combinacao title + date nao existia antes.


CUIDADOS IMPORTANTES

1. Evitar spam em importacao/restauracao

O painel tem importacao de JSON e restauracao do conteudo original. Se a notificacao disparar para qualquer item novo, uma importacao grande pode mandar varias mensagens no Discord.

Opcoes:
- limitar notificacoes a itens criados individualmente no painel;
- ou manter notificacao em importacao, mas com limite;
- ou enviar uma mensagem resumida: "Foram adicionadas X sessoes e Y notas".

A melhor escolha para este projeto provavelmente e:
- disparar notificacoes para novos itens normalmente;
- se houver mais de 3 itens novos no mesmo salvamento, mandar uma unica mensagem resumida.

2. Evitar notificacao duplicada

Se o usuario salvar a mesma sessao varias vezes, nao deve notificar novamente.

Comparar conteudo anterior x conteudo novo resolve isso.

3. Webhook pode falhar

Se o Discord estiver fora ou o webhook estiver errado, o salvamento do conteudo nao deveria falhar.

Recomendacao:
- salvar no Supabase primeiro;
- tentar enviar notificacao depois;
- se a notificacao falhar, registrar console.error, mas retornar sucesso para o painel.

4. Variavel de ambiente ausente

Se DISCORD_WEBHOOK_URL nao estiver configurada, o app deve simplesmente nao enviar notificacao.

Isso permite rodar localmente sem Discord.

5. Seguranca

O webhook permite publicar no canal. Quem tiver essa URL consegue mandar mensagem no Discord.

Por isso:
- nao colocar em VITE_;
- nao colocar no frontend;
- nao commitar em arquivo .env;
- salvar apenas nas variaveis de ambiente da Vercel/Supabase/local.


IMPLEMENTACAO RECOMENDADA

Criar helpers em campaign.functions.ts ou em um novo arquivo server-only, por exemplo:

src/lib/discord.server.ts

Funcoes sugeridas:

- getDiscordWebhookUrl()
- notifyDiscord(message)
- findNewSessions(previousContent, nextContent)
- findNewMasterNotes(previousContent, nextContent)
- notifyCampaignAdditions(previousContent, nextContent)

No saveCampaignContent:

1. requireAdmin()
2. readCampaignContent(false)
3. normalizeContent(data)
4. saveCampaignContentToDb(content)
5. notifyCampaignAdditions(previousContent, content)
6. return sucesso


EXEMPLO DE PAYLOAD DISCORD

POST para o webhook:

{
  "content": "Novo resumo de sessao adicionado: Sessao 04 - O Caminho Sob a Neve\nhttps://seu-site.com/sessoes/sessao-04-o-caminho-sob-a-neve"
}

Ou usando embeds:

{
  "embeds": [
    {
      "title": "Novo resumo de sessao",
      "description": "Sessao 04 - O Caminho Sob a Neve",
      "url": "https://seu-site.com/sessoes/sessao-04-o-caminho-sob-a-neve",
      "color": 12763842
    }
  ]
}

Embeds ficam mais bonitos no Discord.


VARIAVEIS RECOMENDADAS

DISCORD_WEBHOOK_URL=
PUBLIC_SITE_URL=https://seu-dominio.com

PUBLIC_SITE_URL seria usada apenas no servidor para montar links absolutos nas mensagens.


CONCLUSAO

E totalmente possivel.

A integracao mais simples, segura e suficiente para este projeto e usar Discord Webhook server-side. A melhor implementacao e detectar novos itens comparando o conteudo anterior com o novo durante o salvamento do painel, salvar no Supabase primeiro e notificar o Discord depois.

Eu recomendo implementar com:
- notificacao individual para 1 a 3 novos itens;
- notificacao resumida quando uma importacao ou restauracao adicionar muitos itens;
- falha do Discord sem bloquear o salvamento;
- webhook configurado apenas em variavel de ambiente do servidor.
